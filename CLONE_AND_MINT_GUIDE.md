# クローンして別の画像でNFTを発行する手順 🚀

このガイドでは、既存のスマートコントラクト（パッケージ）を再利用して、画像や名前だけが異なる新しいNFTを発行する最短手順を説明します。
**プログラム（Moveコード）の修正や再デプロイは不要です。**

---

## 📋 事前準備
1. **GitHubからクローン**:
   ```bash
   git clone [リポジトリのURL]
   cd meow_nft
   ```
2. **新しい画像の準備**:
   - [Pinata](https://www.pinata.cloud/) 等に画像をアップロードし、新しい **CID** を取得します。
   - URL形式: `ipfs://[新しいCID]`

---

## 🛠️ 発行手順

### 1. ローカル設定ファイルの作成
リポジトリ直下に `config.local` という名前でファイルを作成します。
(このファイルは `.gitignore` に登録されているため、GitHubにプッシュされる心配はありません)

```bash
touch config.local
```

### 2. 設定の書き込み
エディタで `config.local` を開き、以下の内容をコピー＆ペーストして、作りたいNFTの情報に書き換えます。

```bash
# パッケージIDは既存のものをそのまま利用（金型を再利用）
PACKAGE_ID="0xa9151794e08d2feb3e1261a2718b42c787cfdeb296ba282dae44aca208957818"

# --- ここに新しい情報を入力 ---
IMAGE_URL="ipfs://[取得した新しいCID]"
NFT_TITLE="[NFTのタイトル]"
NFT_DESC="[NFTの説明文]"
```

### 3. 発行（ミント）の実行
スクリプトに実行権限を与えてから、配布先のアドレスを指定して実行します。

```bash
# 実行権限の付与（初回のみ）
chmod +x mint_nft.sh

# 実行
./mint_nft.sh [相手のウォレットアドレス]
```

---

## 💡 知っておくと便利なこと

### なぜパッケージIDを変えなくていいの？
パッケージIDは**「NFTを作るための金型（工場）」**のようなものです。
一度デプロイしてしまえば、その「金型」に流し込むデータ（画像や名前）を変えるだけで、無限にバリエーション豊かなNFTを作ることができます。

### Object ID との関係
同じパッケージIDを使っても、発行されるたびに各NFTには固有の **Object ID** が割り振られます。そのため、以前に発行したNFTが上書きされたり消えたりすることはありません。

### 情報の秘匿
画像CIDや配布先の情報は `config.local` に書くことで、リポジトリの履歴に残さず安全に作業できます。
